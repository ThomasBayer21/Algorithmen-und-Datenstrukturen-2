Was ist eine Hashtabelle? Eine Hashtabelle speichert Schlüssel an einem durch eine Hashfunktion berechneten Index. Bei einer Kollision (zwei Schlüssel landen am gleichen Index) gibt es zwei Strategien: Verkettung – jeder Platz verweist auf eine Liste; Offene Adressierung – der nächste freie Platz wird durch Sondierung gesucht (linear, quadratisch, doppeltes Hashing). Der Ladefaktor α = m/N (belegte Plätze / Tabellengröße) bestimmt die Leistung. Löschen bei offener Adressierung erfordert eine Tombstone-Markierung (✗) – der Platz wird als „früher belegt" markiert, damit spätere Suchpfade nicht unterbrochen werden.
